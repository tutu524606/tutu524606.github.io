# 背景

  • 在网页中浏览器资源开销最大便是DOM节点了，DOM很慢并且很庞大，网页性能都是由JavaScript修改DOM所引起的。我们使用JavaScript来操作DOM，操作效率往往很低，由于DOM被表示为树结构，每次DOM中的某些内容都会发生变化，因此对DOM的更改非常快，但更改后的元素，并且它的子项必须经过Reflow / Layout阶段，然后浏览器必须重新绘制更改，这是很慢的。因此，回流/重绘的次数越多，应用程序就越卡顿。但是，JavaScript运行速度很快，虚拟DOM是放在JavaScript和HTML中间的一个层。它可以通过新旧DOM的对比，来获取对比之后的差异对象，然后有针对性的把差异部分真正地渲染到页面上，从而减少实际DOM操作，最终达到性能优化的目的。

# 虚拟 DOM 原理流程

  • 简单概括一下三点：
  1. 用JavaScript模拟 DOM 树，并渲染 DOM 树
  2. 比较新旧 DOM 树，得到比较后的差异对象
  3. 把差异对象绘制到渲染的 DOM 树中

# 虚拟 DOM 类

class CreateEl{
    constructor(tagName, props, children){
        // 只传入两个参数
        if(Array.isArray(props)){
            children = props;
            props = {};
        }
        // tagName props children 数据保存到this对象上
        this.tagName = tagName;
        this.props = props || {};
        this.children = children || [];
        this.key = props ? props.key : undefined;

        let count = 0;
        // 遍历children数组，如果不是CreateEl的实例对象，则转成字符串
        this.children.forEach(child => {
            if(child instanceof CreateEl){
                count += child.count;
            }else{
                child = '' + child;
            }
            // 给每一个节点设置一个count
            this.count++;
        })
    }
    // 构建一个 DOM 树：创建节点，设置节点属性，递归创建 DOM 树并追加到页面结构中
    render(){
        const el = document.createElement(this.tagName);
        const props = this.props;
        for(let [key,val] of Object.entries(props)){
            el.setAttribute(key,val);
        }
        this.children.forEach(child => {
            let childEl = (child instanceof CreateEl) ? child.render() : document.createTextNode(child);
            el.appendChild(childEl);
        })
        return el;
    }
}

const createEl = (tagName, props, children) => new CreateEl(tagName, props, children);
const vdom = create('div',{'id': 'box'},[
    createEl('h1', {style: 'color: pink'}, ['I am h1']),
    createEl('ul',{class: 'list'},[
        createEl('li', ['#list1']),
        createEl('li', ['#list2']),

const rootnode = createEl.render();
document.appendChild(rootnode);

# 虚拟 DOM diff算法

  • 上面已经创建了一个 DOM 树，然后再创建不同的 DOM 树做比较，得到比较的差异对象，是虚拟 DOM 中的最核心部分，也就是常说的虚拟 DOM 的 diff 算法，两棵完全的树比较的时间复杂度为O（n^3）。但是在web中很少用到跨层级 DOM 树的比较，所以一个层级跟一个层级对比，复杂度可以达到O（n）。如下图：

节点间的差异，总结一下4种：

  1. 直接替换原有节点
  2. 调整子节点，包括移动、删除、替换、添加
  3. 修改节点属性
  4. 修改节点文本内容
